# 03 · Website

## Purpose

Describe what the PYRAXIS website is, what it must communicate, and the business logic behind it.

## Responsibilities

Bridge between brand/product intent (`02-philosophy.md`, `07-brand.md`) and the concrete scene-by-scene execution in `03-website.md` (this file) and the animation system (`08-animation-system.md`).

## Core Principles

**What is being sold.** PYRAXIS constructs growth infrastructure: interconnected systems that convert visitors → customers → advocates → compounding revenue. One integrated system, not a service menu. (Product Bible §01)

**The business problem addressed.** Businesses run broken systems: ghosted leads (lost revenue), manual follow-up (wasted time), slow response (lost opportunities), no retention system (churn), no referral loop (stagnant growth). These are one broken infrastructure, not separate problems. (Product Bible §02)

**The system, visualized as flow:**
```
Website → Lead → Booking → Follow-up → Retention → Review → Referral → Intelligence
```
Every node connects; every signal flows; the system never sleeps. (Product Bible §03)

**Growth Engines** — the deployable modules, all sharing one signal system:

| Engine | Function | Outcome |
|---|---|---|
| Lead Engine | Capture/qualify inbound signals | Qualified customers |
| Booking Engine | Convert interest to commitment | Zero missed opportunities |
| Follow-up Engine | Automated nurture | Deals that would otherwise die |
| Retention Engine | Post-purchase engagement | Repeat revenue |
| Review Engine | Reputation capture | Customers become marketing |
| Referral Engine | Referral activation | Compounding growth |
| Intelligence Layer | Cross-engine data synthesis | Decisions from signal, not guesswork |

(Product Bible §04)

**The Signal** is the conceptual hero of the product: one customer's journey through the whole infrastructure (Visitor → Lead → Booking → Customer → Review → Referral → Lifetime Value → Intelligence). The site's central animated motif (see `09-particle-engine.md`, Design System §20) exists so the visitor *watches* infrastructure work rather than reading about it. (Product Bible §05)

**Customer journey (business view):** Awareness → Capture → Conversion → Delivery → Scale → Intelligence. (Product Bible §06)

**Deployment philosophy:** PYRAXIS deploys complete infrastructure, never isolated components: Discovery → Strategy → Build → Launch → Scale. No arrows shown on the website — the Signal travels through each stage instead. (Product Bible §07)

**Pricing philosophy:** No listed prices. Pricing is framed as infrastructure investment: "what does a broken system cost you monthly?" vs. "what does a working system produce monthly?" — the delta is the value. (Product Bible §08)

**Portfolio philosophy:** Only real work, never fake case studies or stock screenshots. Until real work exists, quality is communicated through engineering and thinking alone. (Product Bible §09)

**Scalability:** Launch (Website + Lead) → Growth (+Booking +Follow-up) → Retention (+Review +Referral) → Scale (+Intelligence) → Enterprise (full ecosystem, custom engines). (Product Bible §10)

**What PYRAXIS explicitly is NOT:** a website agency, an AI agency, an automation agency, or freelance service — each rejected as too narrow, trend-dependent, feature-focused, or transactional respectively. (Product Bible §12)
